## 1. Motivation

Phase 1 upgrades the data tier to unblock Phase 3 AI features and clear an SSPL licensing risk against ~$200K+ ARR in active enterprise procurement review. See [PRD §1](./1.0_prd_infrastructure-upgrades.md) and [BRD §3](./1.0_brd_infrastructure-upgrades.md).

Each dependency in this bundle is justified by a specific functional requirement:

- **PostgreSQL 17** → FR-01/FR-02/FR-03 (EOL runway, `pg_stat_statements`)
- **pgvector** → FR-04/FR-05 (co-located SQL+vector queries; new capability)
- **Valkey or Redis 8** → FR-06/FR-07, NFR-06 (off SSPL)
- **qdrant-client 1.12.x + Qdrant server 1.12+** → FR-08/FR-09/FR-10/FR-11 (named & sparse vectors)
- **Neo4j 5.x pinned patch + APOC + GDS** → FR-12 (no silent plugin drift)

**Triggering events**:
- SSPL flagged by 2 enterprise prospects (procurement blocker)
- PostgreSQL 15 community EOL Nov 2027
- Phase 3.6 hybrid search requires Qdrant named/sparse vectors
- Phase 3.5 embedding pipeline requires pgvector

**Tracking issue**: [PHASE1-001](https://github.com/legalai/platform/issues/PHASE1-001)

## 7. Maintenance Plan

| Aspect | Detail |
|--------|--------|
| Update strategy | Renovate bot opens PRs for patch versions weekly; minor versions reviewed monthly; majors require a new Dep Review |
| Update owner | Platform team (rotating on-call); pgvector and Qdrant tracked by Backend AI subteam |
| Breaking change policy | Pin to compatible release (`~=X.Y`); never auto-merge minors; majors gated by Dep Review + Tech Spec amendment |
| Deprecation monitoring | Subscribe to GitHub releases for: postgres, pgvector, valkey, qdrant, qdrant-client, neo4j; route alerts to `#eng-platform` |
| Fallback if abandoned — pgvector | Fork is trivial (PG extension, ~5K LOC, PG license). Alternative: `pgvecto.rs`. |
| Fallback if abandoned — Valkey | Switch to Redis 8 OSS (wire compatible) or KeyDB |
| Fallback if abandoned — Qdrant | Migration path to Weaviate or Milvus is large; mitigated by Qdrant's corporate funding and multi-cloud adoption |
| Fallback if abandoned — Neo4j | Migration to Memgraph (Cypher-compatible); large effort, low likelihood |

## 8. Alternatives Considered

| Alternative | Why Not Chosen |
|-------------|---------------|
| Stay on PG 15 + Redis 7 + Qdrant 1.7 | Fails PRD FR-06, leaves SSPL exposure, blocks Phase 3 (5 blockers per PRD §12) |
| Replace Qdrant with pgvector entirely | pgvector lacks named & sparse vectors required by Phase 3.6 (PRD §2) |
| KeyDB instead of Valkey | Smaller community, weaker corporate backing post-Snap acquisition pause |
| Redis 8 OSS over Valkey | AGPLv3 ambiguity vs Valkey BSD-3; deferred to ADR-001 |
| `pgvecto.rs` over pgvector | Smaller ecosystem, less production usage at our scale; kept as fallback |
| Neo4j 6 (when GA) | Not GA on Phase 1 timeline; would force a separate Dep Review |

## 9. Rollback Plan

Each dependency rolls back independently via image tag revert. NFR-08 requires < 15 min per rollback.

```bash
# Roll back PostgreSQL
git revert <PR-1.1.1-sha>
docker compose down postgres
docker volume rm legalai_pg_data    # dev/CI only
docker compose up -d postgres
alembic upgrade head

# Roll back qdrant-client
pip install 'qdrant-client==1.7.0'
poetry lock --no-update
git revert <PR-1.4.1-sha>

# Roll back Qdrant server
docker compose down qdrant
# WARNING: 1.12 → 1.7 is not supported; rebuild from empty volume in dev/CI
docker volume rm legalai_qdrant_data
docker compose up -d qdrant

# Roll back cache backend
git revert <PR-1.3.2-sha>
docker compose up -d redis

# Roll back Neo4j pin
git revert <PR-1.5.1-sha>
```

**Tracking issue for rollback verification**: [PHASE1-099](https://github.com/legalai/platform/issues/PHASE1-099)

Production rollback procedure is documented separately in the [Migration Guide](./1.0_mig-guide_infrastructure-upgrades.md) and [Runbook](./1.0_runbook_infrastructure-deployment.md) (both pending per PRD §15).
