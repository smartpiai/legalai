> Phase 1 is an infrastructure phase: per the [Tech Spec §5](./1.0_tech-spec_infrastructure-upgrades.md#5-api-changes), there are **no public HTTP API surface changes**. This ID Spec therefore defines the **internal Python interfaces** introduced or refactored in Phase 1 — the contracts between `services/embedding.py` (consumer) and the new `repositories/EmbeddingRepository` and refactored `core/vector_store` (producers). HTTP/SSE/WebSocket sections are marked `N/A`.

| Role | Component |
|------|-----------|

This spec freezes the Python signatures, DTO shapes, error taxonomy, and tenant-isolation contract for the embedding data-access layer so that Phase 2 work can proceed against a stable interface.

## 2. Global API Policies

N/A — no HTTP surface in Phase 1. The equivalent **internal call policies** are:

| Policy | Value | Notes |
|--------|-------|-------|
| Transport | In-process Python (`async def`) | All methods are `async`; never call from a sync context |
| Tenant scoping | **Mandatory** — `tenant_id: UUID` injected at repository construction | Enforced in every `WHERE` clause; no method takes `tenant_id` as a per-call argument |
| Session lifetime | One `AsyncSession` per request, owned by FastAPI dependency | Repository must not commit; caller owns the unit-of-work |
| Default timeout | Inherits SQLAlchemy pool timeout (30 s) | k-NN methods should complete < 50 ms p95 (NFR-01) |
| Max batch size | `insert_many`: 1000 rows; `knn`: `k ≤ 200` | Enforced via `Field(le=...)` validators |
| Concurrency | Safe for concurrent calls across distinct sessions; **not** safe to share an `AsyncSession` across tasks | Standard SQLAlchemy 2.0 contract |

### Rate Limiting Strategy

N/A — internal interface. Backpressure is the SQLAlchemy connection pool (`pool_size=20, max_overflow=10`, per Tech Spec §3 Concurrency).

## 4. Pagination

Cursor-based pagination for `EmbeddingRepository.list_by_document` (used by admin/debug paths and future re-embedding workflows).

```python
class EmbeddingPage(BaseModel):
    data: list[EmbeddingHit]
    next_cursor: str | None = None
    has_more: bool
```

**Cursor encoding**: opaque base64 of `{"id": "<uuid>", "chunk_index": <int>}`. Stable across calls; clients must treat it as opaque.

**Defaults**: `limit=50`, max `limit=200`. k-NN results (`knn()`) are not paginated — `k` is the explicit cap.

## 7. Data Models / Schemas

### `DocumentEmbedding` (ORM model — `app/models/document_embedding.py`)

Defined in [Tech Spec §3 Key Components](./1.0_tech-spec_infrastructure-upgrades.md#documentembedding-model-new--appmodelsdocument_embeddingpy). Reproduced here as the source of truth for field types:

| Field | Type | Constraints |
|-------|------|-------------|
| `id` | `UUID` | PK, default `uuid4` |
| `tenant_id` | `UUID` | NOT NULL, indexed |
| `document_id` | `UUID` | FK → `documents.id`, ON DELETE CASCADE |
| `chunk_index` | `int` | NOT NULL, ≥ 0 |
| `embedding` | `Vector(1536)` | NOT NULL, cosine distance via HNSW |
| `meta` | `JSONB` | default `{}` |
| `created_at` | `timestamptz` | server default `now()` |
| **Unique** | `(tenant_id, document_id, chunk_index)` | enables idempotent re-embed |

### `EmbeddingInsert` (Pydantic DTO — input)

```python
from uuid import UUID
from pydantic import BaseModel, Field, ConfigDict

class EmbeddingInsert(BaseModel):
    document_id: UUID
    chunk_index: int = Field(..., ge=0)
    embedding: list[float] = Field(..., min_length=1536, max_length=1536)
    meta: dict = Field(default_factory=dict)

    model_config = ConfigDict(
        json_schema_extra={
            "example": {
                "document_id": "9b8e7c54-1f3a-4d2b-9c1e-2a4b6c8d0e12",
                "chunk_index": 0,
                "embedding": [0.0123, -0.0456, "... (1536 floats)"],
                "meta": {"page": 1, "section": "Indemnification"}
            }
        }
    )
```

### `EmbeddingHit` (Pydantic DTO — output)

```python
class EmbeddingHit(BaseModel):
    id: UUID
    document_id: UUID
    chunk_index: int
    score: float = Field(..., description="Cosine similarity in [0, 1]; higher = more similar")
    meta: dict

    model_config = ConfigDict(
        json_schema_extra={
            "example": {
                "id": "1c2d3e4f-5a6b-7c8d-9e0f-1a2b3c4d5e6f",
                "document_id": "9b8e7c54-1f3a-4d2b-9c1e-2a4b6c8d0e12",
                "chunk_index": 0,
                "score": 0.9213,
                "meta": {"page": 1, "section": "Indemnification"}
            }
        }
    )
```

### `KnnQuery` (Pydantic DTO — input)

```python
class KnnQuery(BaseModel):

    k: int = Field(default=10, ge=1, le=200)

```

## 8. Tool Interfaces (repurposed: Internal Method Contracts)

> Phase 1 has no Agent SDK tools. This section documents the canonical Python method signatures that consumers may call. Each entry is the equivalent of an "endpoint" for an internal interface.

### `EmbeddingRepository.__init__`

```python
def __init__(self, session: AsyncSession, tenant_id: UUID) -> None: ...
```

**Contract**: `tenant_id` is bound for the lifetime of the repository instance and is injected into every query. Constructing without a tenant raises `ValueError`.

---

### `EmbeddingRepository.insert`

**Description**: Insert one embedding row. Idempotent on `(tenant_id, document_id, chunk_index)`.

**Signature:**
```python
async def insert(self, payload: EmbeddingInsert) -> UUID: ...
```

**Returns**: The `id` of the inserted (or pre-existing) row.

**Behavior**:
- Uses `INSERT ... ON CONFLICT (tenant_id, document_id, chunk_index) DO NOTHING RETURNING id`.
- On conflict, performs a follow-up `SELECT id` so the return value is always populated.
- Does **not** commit — caller owns the transaction.

**Example call:**
```python
repo = EmbeddingRepository(session, tenant_id=current_tenant.id)
new_id = await repo.insert(EmbeddingInsert(
    document_id=doc.id, chunk_index=0, embedding=vec, meta={"page": 1}
))
```

**Errors:**

| Exception | When |
|-----------|------|
| `ValueError` | `embedding` not exactly 1536 dims (caught by Pydantic before reaching DB) |
| `ForeignKeyViolation` | `document_id` does not exist (or belongs to a different tenant — caller must pre-check) |
| `IntegrityError` | Concurrent insert with mismatched embedding bytes (treated as fatal — caller should not retry) |
| `OperationalError` | Transient DB error — caller may retry once |

---

### `EmbeddingRepository.insert_many`

**Description**: Bulk insert with the same idempotency semantics.

**Signature:**
```python
async def insert_many(self, payloads: list[EmbeddingInsert]) -> list[UUID]: ...
```

**Constraints**: `len(payloads) ≤ 1000`. Larger batches raise `ValueError` — caller must chunk.

---

### `EmbeddingRepository.knn`

**Description**: k-nearest-neighbour search over the current tenant's embeddings.

**Signature:**
```python
async def knn(self, query: KnnQuery) -> list[EmbeddingHit]: ...
```

**Behavior**:
- SQL: `SELECT ... FROM document_embeddings WHERE tenant_id = :t [AND meta @> :f] ORDER BY embedding <=> :q LIMIT :k`.
- `score` returned to the consumer is `1 - cosine_distance`, normalized to `[0, 1]`.
- Index hint: HNSW (`idx_doc_emb_hnsw`) is selected by the planner; verified via `EXPLAIN` in tests.
- Performance contract: p95 < 50 ms at 100K rows, k=10 (PRD NFR-01).

**Example call:**
```python
hits = await repo.knn(KnnQuery(
    query_vector=embed("indemnification clauses"),
    k=10,
    filters={"contract_type": "MSA"},
))
```

**Example return:**
```python
[
    EmbeddingHit(id=..., document_id=..., chunk_index=0, score=0.9213, meta={...}),
    EmbeddingHit(id=..., document_id=..., chunk_index=3, score=0.8871, meta={...}),
    # ... up to k entries
]
```

**Errors:**

| Exception | When |
|-----------|------|
| `ValueError` | `k` out of range, or vector dim ≠ 1536 (Pydantic-enforced) |
| `OperationalError` | DB unreachable / timeout — transient, retryable |
| `TenantIsolationError` (custom) | Reserved — raised if a tests-only assertion catches a missing `tenant_id` clause |

---

### `EmbeddingRepository.delete_by_document`

**Signature:**
```python
async def delete_by_document(self, document_id: UUID) -> int: ...
```

**Returns**: Count of rows deleted. Always tenant-scoped — deletes nothing if `document_id` belongs to another tenant.

---

### `EmbeddingRepository.list_by_document`

**Signature:**

```python
async def list_by_document(
```

### `vector_store.create_collection` (refactored)

**Description**: Create a Qdrant collection supporting multiple named dense vectors and optional sparse vectors. Idempotent.

**Signature:**
```python
from qdrant_client.http.models import VectorParams, SparseVectorParams

async def create_collection(
    name: str,
    dense_vectors: dict[str, VectorParams],
    sparse_vectors: dict[str, SparseVectorParams] | None = None,
) -> None: ...
```

**Backward-compat shim**: A legacy positional call `create_collection(name, size=1536, distance=Distance.COSINE)` is accepted and internally rewritten to `dense_vectors={"default": VectorParams(1536, COSINE)}`. The shim emits a `DeprecationWarning` and is scheduled for removal in Phase 3 (see §10 Deprecation Flow).

**Example call:**
```python
await create_collection(
    name="contracts_v2",
    dense_vectors={
        "text": VectorParams(size=1536, distance=Distance.COSINE),
    },
    sparse_vectors={
        "bm25": SparseVectorParams(),
    },
)
```

**Errors:**

| Exception | When |
|-----------|------|
| `qdrant_client.http.exceptions.UnexpectedResponse` | Server rejects schema (e.g., name conflict with mismatched config) |
| `ValueError` | `dense_vectors` is empty |
| `ConnectionError` | Qdrant unreachable — transient |

**Idempotency**: If a collection with `name` already exists *and* its schema matches the requested dense+sparse config, the call is a no-op. If the schema differs, raises `UnexpectedResponse` (caller decides to drop/recreate).

## 10. Versioning & Compatibility

### Versioning Strategy

| Aspect | Approach |
|--------|----------|
| Versioning method | Python package semver via `backend/pyproject.toml`. Internal modules follow the repo version. |
| Breaking change definition | Removing a method, changing a parameter type, changing return shape, removing a Pydantic field, tightening a validator |
| Non-breaking changes | Adding optional kwargs, adding new methods, adding optional Pydantic fields, loosening validators |

### Version History

| Version | Changes | Backward Compatible |
|---------|---------|---------------------|
| 1.0 | Initial: `EmbeddingRepository`, `EmbeddingInsert/Hit/KnnQuery/Page` DTOs, refactored `vector_store.create_collection` with named + sparse vectors and legacy positional shim | N/A |

### Deprecation Flow

| Step | Action | Timeline |
|------|--------|----------|
| 1 | Mark legacy positional `create_collection(name, size, distance)` shim with `DeprecationWarning` | Phase 1 (PR 1.4.2) |
| 2 | Add `# DEPRECATED:` comment with removal target (Phase 3) | Phase 1 |
| 3 | Grep CI check that no in-repo callers use the legacy form | Phase 2 entry |
| 4 | Remove shim | Phase 3 |
