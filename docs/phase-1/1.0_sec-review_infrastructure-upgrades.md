## 1. Change Summary

Phase 1 is an **infrastructure-only upgrade**. No new HTTP endpoints, no new LLM invocation paths, no new vendors.

Changes in scope:
- PostgreSQL **15 → 17**, image switched to `pgvector/pgvector:pg17`; `pgvector` and `pg_stat_statements` extensions enabled in `scripts/init-postgres.sql`.
- Redis 7 → **Valkey 8** (`valkey/valkey:8-alpine`), wire-compatible drop-in.
- Qdrant **1.7 → 1.12**; `qdrant-client` Python package bumped across 5 minor versions.
- Neo4j pinned at **5.20.0** with APOC + GDS plugins.
- New Alembic migration `009` introducing `document_embeddings(tenant_id, document_id, chunk_index, embedding vector(1536), meta jsonb)` and `EmbeddingRepository` (knn / insert / delete_by_document), all tenant-scoped at the application layer.
- New Prometheus metric `pgvector_knn_duration_seconds` and structured log event `embedding.repo.knn`.

Net effect: HTTP attack surface unchanged; **data-layer** attack surface materially changes (new table holding PII-equivalent embeddings, two new container supply-chain anchors, one new C extension loaded into PG, expanded extension surface via `pg_stat_statements`).

### Threat Scenarios

#### T1: Cross-tenant read of `document_embeddings` via missing RLS

| Attribute | Value |
|-----------|-------|
| **Attack vector** | A code path (current or future) issues a query against `document_embeddings` without injecting `WHERE tenant_id = :t` — e.g. a maintenance script, a future repository method, an ORM `.all()` call, or SQL via `pg_stat_statements`-leaked patterns. CLAUDE.md mandates Postgres RLS as the isolation mechanism; migration `009` does not enable it. |
| **Likelihood** | 4 |
| **Impact** | 5 |
| **Risk Score** | **20 (Critical)** |

**Mitigations**:
- Enable RLS on `document_embeddings` in migration `009`: `ENABLE ROW LEVEL SECURITY` + `CREATE POLICY tenant_isolation USING (tenant_id = current_setting('app.tenant_id')::uuid)`.
- Set the `app.tenant_id` GUC in the `AsyncSession` dependency on every request.
- Integration test: open a session as tenant A, attempt raw SQL `SELECT ... FROM document_embeddings WHERE tenant_id = '<tenant_B>'`, assert zero rows.

**Residual risk**: Privileged maintenance roles bypass RLS — accepted; mitigated by audit logging on the admin role.

#### T2: PII spillage into unconstrained `meta` JSONB

| Attribute | Value |
|-----------|-------|
| **Attack vector** | `meta JSONB` has no schema. A well-intentioned developer stores `{"clause_text": "...", "party_name": "..."}` for debugging convenience. That content is then visible via `pg_stat_statements` predicates, log correlation, and any future analytics export — none of which are governed as PII paths. |
| **Likelihood** | 4 |
| **Impact** | 4 |
| **Risk Score** | **16 (High)** |

**Mitigations**:
- Define a closed allowlist of `meta` keys: `{page, section, char_start, char_end, content_hash}`.
- Enforce in `EmbeddingRepository.insert` via Pydantic model + reject unknown keys.
- Add a CHECK constraint: `CHECK (jsonb_typeof(meta) = 'object' AND NOT (meta ? 'text') AND NOT (meta ? 'clause_text'))` as a defense-in-depth tripwire.

**Residual risk**: Determined developer can still encode text into allowed numeric fields; mitigated by code review.

#### T3: Tenant enumeration via Prometheus `tenant_id` label

| Attribute | Value |
|-----------|-------|
| **Attack vector** | `pgvector_knn_duration_seconds{tenant_id="..."}` is scraped into Prometheus. Anyone with read access to the metrics backend (operators, on-call, dashboards shared with vendors) can enumerate the full tenant set, count their queries, and infer activity patterns. Also produces unbounded metric cardinality → Prometheus DoS. |
| **Likelihood** | 5 |
| **Impact** | 3 |
| **Risk Score** | **15 (High)** |

**Mitigations**:
- Drop the `tenant_id` label from the metric. Move to a bounded dimension (e.g. `tenant_tier` or a hashed bucket of N=16).
- Keep `tenant_id` in OTel **trace attributes** instead — those flow to an access-controlled trace store.

**Residual risk**: Aggregate query volume still observable; acceptable.

#### T4: Embedding inversion → contract content reconstruction

| Attribute | Value |
|-----------|-------|
| **Attack vector** | An attacker who exfiltrates embeddings (DB dump, backup leak, compromised volume) runs a published embedding-inversion attack (e.g. `vec2text`) against the `text-embedding-ada-002` vectors and reconstructs material chunks of contract text. Volume-level encryption does not help if the dump is taken from the live DB. |
| **Likelihood** | 2 |
| **Impact** | 5 |
| **Risk Score** | **10 (Medium)** |

**Mitigations**:
- Classify `embedding` columns as PII-equivalent in the data inventory.
- Restrict SELECT on `document_embeddings` to the application role only; deny `pg_read_all_data`.
- Track as input to a future Phase that adds column-level encryption / KMS-backed envelope for embeddings.

**Residual risk**: Live application role compromise still leaks vectors; mitigated by F1 (RLS) and credential rotation.

#### T5: Supply-chain compromise of new container images

| Attribute | Value |
|-----------|-------|
| **Attack vector** | `pgvector/pgvector:pg17` and `valkey/valkey:8-alpine` are new publishers in the trust graph. A tag-mutation attack (or upstream compromise) replaces the image with a backdoored variant; next deploy pulls it. |
| **Likelihood** | 2 |
| **Impact** | 5 |
| **Risk Score** | **10 (Medium)** |

**Mitigations**:
- Pin all four images by **digest** (`@sha256:…`), not tag.
- Add Trivy/Grype scan to CI for each image; record SBOM.
- Allowlist registries; disable pulling from unverified sources.

**Residual risk**: Compromise at the moment of digest pinning; mitigated by SBOM diff review.

#### T6: APOC procedure abuse on Neo4j

| Attribute | Value |
|-----------|-------|
| **Attack vector** | APOC ships `apoc.load.json`, `apoc.export.csv.*`, `apoc.load.jdbc`, etc. If `apoc.import.file.enabled` / `apoc.export.file.enabled` are not explicitly disabled, any user/code with Cypher execution can read or write the Neo4j container filesystem and make outbound HTTP calls — turning Neo4j into an SSRF/exfiltration pivot. |
| **Likelihood** | 3 |
| **Impact** | 4 |
| **Risk Score** | **12 (Medium)** |

**Mitigations**:
- Set `NEO4J_apoc_import_file_enabled=false`, `NEO4J_apoc_export_file_enabled=false`.
- Set `NEO4J_dbms_security_procedures_unrestricted` to a minimal explicit allowlist.
- Validate in PR 1.5.2 smoke test by attempting `CALL apoc.export.csv.all(...)` and asserting failure.

**Residual risk**: GDS procedures still expose compute amplification; acceptable.

#### T7: Pre-existing `vector_store.search_vectors()` lacks tenant filter

| Attribute | Value |
|-----------|-------|
| **Attack vector** | Phase 1 refactors `core/vector_store.py` for named vectors. The unfiltered `search_vectors()` variant predates Phase 1 but is now adjacent to changed code. Any caller invoking it bypasses the tenant filter that `search_similar_contracts()` / `search_by_metadata()` apply at the app layer (Qdrant itself does not enforce). |
| **Likelihood** | 3 |
| **Impact** | 5 |
| **Risk Score** | **15 (High)** |

**Mitigations**:
- Deprecate `search_vectors()`; make `tenant_id` a required argument or remove the symbol.
- Add a CI grep check: `grep -R "search_vectors(" backend/app | grep -v "tenant_id="` must return zero hits.

**Residual risk**: Low after enforcement.

#### T8: `pg_stat_statements` leakage of JSONB predicates

| Attribute | Value |
|-----------|-------|
| **Attack vector** | `pg_stat_statements` normalizes SQL but does not normalize JSONB path predicates the same way; `meta @> '{"party_name":"Acme"}'`-style queries can leak literals to any role with `pg_read_all_stats`. |
| **Likelihood** | 2 |
| **Impact** | 3 |
| **Risk Score** | **6 (Medium)** |

**Mitigations**:
- Restrict `pg_read_all_stats` membership to the admin role only.
- Combined with F2 allowlist, forbid JSONB path predicates against `meta` outside the repository.

**Residual risk**: Admin role compromise; accepted.

## 13. Incident Response

| Aspect | Detail |
|--------|--------|
| IR playbook | Use general IR process; this section adds infra-specific detection/containment |
| Detection | (a) Alert on `document_embeddings` query with NULL `tenant_id` filter (pg audit extension or app-side log); (b) Prometheus alert on metric cardinality explosion (proxy for F3 misuse); (c) Trivy scheduled re-scan of pinned digests; (d) Neo4j query log alert on `apoc.export.*` / `apoc.load.*` invocations |
| Containment | (a) Revoke `app` role SELECT on `document_embeddings` (kill switch — embedding features degrade gracefully); (b) Disable Valkey via compose down + standby; (c) Roll back to previous pinned digests via `docker-compose.override.yml`; (d) `pg_stat_statements_reset()` to clear leaked literals |
| Notification | Security Slack channel; affected tenants within 72h per GDPR Art. 33 if cross-tenant read confirmed |
| Evidence preservation | Snapshot PG volume; export `pg_stat_statements`; preserve container image digests; export Prometheus & trace data for the incident window |
