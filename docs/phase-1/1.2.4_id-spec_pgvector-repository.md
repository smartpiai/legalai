## 1. Overview

Defines the contract for `PgVectorRepository`, the data-access layer that wraps the `document_embeddings` table introduced in PR 1.2.1. It is the *only* sanctioned path for application code to read/write embeddings in PostgreSQL; services and workers MUST NOT issue raw SQL against `document_embeddings`.

| Role | Component |
|------|-----------|
| **Producer** | `backend/app/repositories/pgvector.py` (this spec) |
| **Consumer (write)** | Embedding worker (`backend/app/workers/embedding_worker.py`) |
| **Consumer (read)** | Retrieval service (`backend/app/services/retrieval.py`), GraphRAG service |
| **Backed by** | `DocumentEmbedding` SQLAlchemy model (PR 1.2.1) on PG 17 + pgvector Ōēź0.7 |

### Goals
- Pin the method signatures, types, and exceptions before consumers are written.
- Enforce tenant scoping at the type level ŌĆö every method takes `tenant_id` as a required argument.
- Standardize on cosine distance (`<=>`) so consumers cannot accidentally request L2/IP.
- Provide a clean error taxonomy (`DimensionMismatch`, `ConstraintViolation`, `TenantMismatch`, `RepositoryError`) so callers can map failures to user-facing behavior without sniffing driver exceptions.

### Non-Goals
- Embedding generation (caller passes pre-computed vectors).
- Hybrid search / metadata filtering beyond `tenant_id` and `document_id` (separate spec).
- Cache layer in front of the repository.
- Cross-store fallback to Qdrant (handled in retrieval service).

## 2. Global API Policies

N/A ŌĆö internal Python interface. Process-local; no transport, CORS, or rate-limiting concerns. Equivalent constraints below:

| Policy | Value | Notes |
|--------|-------|-------|
| Module path | `app.repositories.pgvector` | |
| Async | All methods are `async`; require an `AsyncSession` from `app.core.db.get_session`. | |
| Transactionality | Repository does **not** commit. Caller owns the transaction (Unit of Work). | Matches existing repos. |
| Tenant enforcement | `tenant_id: UUID` is a required positional argument on every method. | Static-checked via mypy. |
| Distance metric | Cosine (`<=>`, `vector_cosine_ops`) ŌĆö hard-coded; not a parameter. | Matches PR 1.2.1 schema. |
| Vector dimensionality | 1536 (validated at the repo boundary; mismatches raise `DimensionMismatch` before hitting PG). | |
| Max batch size (`upsert_embeddings`) | 1000 rows per call. | Larger batches ŌåÆ caller chunks. |
| Max `top_k` (`search_similar`) | 100. | Server-side guard. |
| Logging | All methods emit a structured log at DEBUG with `tenant_id`, method, row count / k. | |
| Tracing | Each method opens a span `repo.pgvector.{method}` with `tenant_id` tag. | |

N/A ŌĆö internal interface. Throughput is governed by the asyncpg pool size and the PG-level targets in the PR 1.2.1 Tech Spec ┬¦9.

### 3a. Method Contracts

#### `__init__(session: AsyncSession) -> None`

**Description**: Construct a repository bound to a SQLAlchemy `AsyncSession`. The session's lifecycle is owned by the caller (typically via `Depends(get_session)`).

**Signature:**
```python
class PgVectorRepository:
    def __init__(self, session: AsyncSession) -> None:
        self._session = session
```

**Preconditions**:
- `session` is an open `sqlalchemy.ext.asyncio.AsyncSession`.
- The bound engine has the `pgvector` type adapter registered (handled by `app.core.db`).

**Errors**: None ŌĆö pure construction.

---

#### `upsert_embeddings(tenant_id, embeddings) -> UpsertResult`

**Description**: Insert a batch of embedding rows for a single tenant. Idempotent on `(document_id, chunk_index, model_name)` via `INSERT ŌĆ” ON CONFLICT DO NOTHING` (matches PR 1.2.1 ┬¦7).

**Signature:**
```python
async def upsert_embeddings(
    self,
    tenant_id: UUID,
    embeddings: Sequence[EmbeddingUpsert],
) -> UpsertResult: ...
```

**Behavior**:
1. Validate `len(embeddings) <= 1000`; otherwise raise `ValueError`.
2. Validate every `embedding.vector` has length 1536; otherwise raise `DimensionMismatch` (no rows written).
3. Validate every `embedding.tenant_id == tenant_id`; otherwise raise `TenantMismatch`.
4. Issue a single `INSERT ŌĆ” ON CONFLICT (document_id, chunk_index, model_name) DO NOTHING RETURNING id`.
5. Return `UpsertResult(inserted=<count of RETURNING rows>, skipped=len(embeddings) - inserted)`.
6. Does **not** call `session.commit()`.

**Input model:**
```python
class EmbeddingUpsert(BaseModel):
    tenant_id: UUID
    document_id: UUID
    chunk_index: int = Field(ge=0)
    vector: list[float] = Field(min_length=1536, max_length=1536)
    model_name: str = Field(min_length=1, max_length=128)

    model_config = ConfigDict(frozen=True)
```

**Return model:**
```python
class UpsertResult(BaseModel):
    inserted: int
    skipped: int  # rows that hit ON CONFLICT DO NOTHING

    @property
    def total(self) -> int:
        return self.inserted + self.skipped
```

**Example:**
```python
result = await repo.upsert_embeddings(
    tenant_id=tenant_id,
    embeddings=[
        EmbeddingUpsert(
            tenant_id=tenant_id,
            document_id=doc_id,
            chunk_index=0,
            vector=[0.0123, -0.0456, ...],  # 1536 floats
            model_name="openai/text-embedding-3-small",
        ),
    ],
)
# result.inserted == 1, result.skipped == 0
```

**Errors**: `DimensionMismatch`, `TenantMismatch`, `ConstraintViolation` (e.g., FK violation when `document_id` does not exist), `ValueError` (batch too large), `RepositoryError` (any other DB-level failure).

#### `search_similar(tenant_id, query_vector, *, top_k=10, document_ids=None, model_name=None) -> list[SimilarChunk]`

**Description**: Cosine-distance ANN search over a tenant's embeddings. Hard-filtered by `tenant_id` so cross-tenant leakage is impossible at the repository layer.

**Signature:**
```python
async def search_similar(
    self,
    tenant_id: UUID,
    query_vector: list[float],
    *,
    top_k: int = 10,
    document_ids: Sequence[UUID] | None = None,
    model_name: str | None = None,
) -> list[SimilarChunk]: ...
```

**Behavior**:
1. Validate `len(query_vector) == 1536`; otherwise raise `DimensionMismatch`.
2. Validate `1 <= top_k <= 100`; otherwise raise `ValueError`.
3. Build query:
   ```sql
   SELECT id, document_id, chunk_index, model_name,
          embedding <=> :q AS distance
     FROM document_embeddings
    WHERE tenant_id = :tenant_id
      [AND document_id = ANY(:document_ids)]
      [AND model_name = :model_name]
    ORDER BY embedding <=> :q
    LIMIT :top_k;
   ```
4. Distance is the raw pgvector cosine distance in `[0, 2]`. Similarity is reported as `1 - distance` for caller convenience.
5. Returns rows in ascending distance order (most similar first).

**Return model:**
```python
class SimilarChunk(BaseModel):
    id: UUID
    document_id: UUID
    chunk_index: int
    model_name: str
    distance: float = Field(ge=0.0, le=2.0)  # cosine distance
    similarity: float = Field(ge=-1.0, le=1.0)  # 1 - distance

    model_config = ConfigDict(frozen=True)
```

**Example:**
```python
hits = await repo.search_similar(
    tenant_id=tenant_id,
    query_vector=q,  # 1536 floats
    top_k=5,
    model_name="openai/text-embedding-3-small",
)
for h in hits:
    print(h.document_id, h.chunk_index, h.similarity)
```

**Errors**: `DimensionMismatch`, `ValueError` (top_k out of range), `RepositoryError`.

**Note on pagination**: ANN search is bounded by `top_k`; cursor pagination is intentionally **not** supported because (a) HNSW does not provide stable ordering across paginated calls and (b) callers needing >100 results should reformulate the query. See ┬¦4.

#### `delete_by_document(tenant_id, document_id, *, model_name=None) -> int`

**Description**: Delete all embedding rows for a single document, optionally narrowed to one model. Used by the document-deletion saga and by the re-embedding workflow.

**Signature:**
```python
async def delete_by_document(
    self,
    tenant_id: UUID,
    document_id: UUID,
    *,
    model_name: str | None = None,
) -> int: ...
```

**Behavior**:
1. Issues `DELETE FROM document_embeddings WHERE tenant_id = :t AND document_id = :d [AND model_name = :m]`.
2. Returns the number of rows deleted.
3. Tenant filter is mandatory and applied **in addition** to the FK CASCADE ŌĆö defense in depth.
4. Does **not** commit.

**Errors**: `RepositoryError`.

---

#### `count_by_tenant(tenant_id, *, model_name=None) -> int`

**Description**: Return the number of embedding rows for a tenant. Used by the `pgvector_rows_per_tenant` gauge (PR 1.2.1 ┬¦10) and by the scale-assumption alert.

**Signature:**
```python
async def count_by_tenant(
    self,
    tenant_id: UUID,
    *,
    model_name: str | None = None,
) -> int: ...
```

**Behavior**:
1. Issues `SELECT COUNT(*) FROM document_embeddings WHERE tenant_id = :t [AND model_name = :m]`.
2. Uses the `idx_document_embeddings_tenant` B-tree.

**Errors**: `RepositoryError`.

### Error Classes

All defined in `app/repositories/pgvector_errors.py`. Hierarchy:

```python
class RepositoryError(Exception):
    """Base for all PgVectorRepository errors. Always carries `tenant_id` if known."""
    def __init__(self, message: str, *, tenant_id: UUID | None = None) -> None:
        super().__init__(message)
        self.tenant_id = tenant_id


class DimensionMismatch(RepositoryError):
    """Raised when a supplied vector is not exactly 1536 dimensions.
    Raised before any DB call ŌĆö no partial writes."""
    def __init__(self, *, expected: int, actual: int, tenant_id: UUID | None = None) -> None:
        super().__init__(
            f"vector dimension mismatch: expected {expected}, got {actual}",
            tenant_id=tenant_id,
        )
        self.expected = expected
        self.actual = actual


class ConstraintViolation(RepositoryError):
    """Raised when a DB constraint fires that the caller could plausibly fix
    (FK violation, NOT NULL, check). Wraps `sqlalchemy.exc.IntegrityError`.
    Note: the unique constraint on (document_id, chunk_index, model_name) does
    NOT raise ŌĆö it is silently absorbed by ON CONFLICT DO NOTHING and reported
    via UpsertResult.skipped."""
    def __init__(self, message: str, *, constraint: str | None = None, tenant_id: UUID | None = None) -> None:
        super().__init__(message, tenant_id=tenant_id)
        self.constraint = constraint


class TenantMismatch(RepositoryError):
    """Raised when a row in the input batch carries a different tenant_id than
    the method argument. Defense in depth against caller bugs that would
    otherwise smuggle cross-tenant data."""
    def __init__(self, *, expected: UUID, actual: UUID) -> None:
        super().__init__(
            f"tenant_id mismatch: method arg {expected}, row {actual}",
            tenant_id=expected,
        )
        self.expected = expected
        self.actual = actual
```

#### Error mapping table

| Driver / SQLAlchemy exception | Repository exception | Notes |
|-------------------------------|----------------------|-------|
| `sqlalchemy.exc.IntegrityError` (FK violation) | `ConstraintViolation(constraint="<fk name>")` | e.g., unknown `document_id`. |
| `sqlalchemy.exc.IntegrityError` (uq_document_embeddings_doc_chunk_model) | **Not raised** ŌĆö absorbed by `ON CONFLICT DO NOTHING`; counted in `UpsertResult.skipped`. | |
| `sqlalchemy.exc.DataError` (`expected 1536 dimensions`) | `DimensionMismatch` | Should be unreachable because the repo validates first; serves as defense in depth. |
| `sqlalchemy.exc.OperationalError`, `DBAPIError`, anything else | `RepositoryError` (re-raised, original chained via `from`) | |

## 10. Versioning & Compatibility

### Versioning Strategy

| Aspect | Approach |
|--------|----------|
| Versioning method | Semantic versioning of the `app.repositories.pgvector` module via the backend package. No URL versioning (internal). |
| Breaking change definition | Removing/renaming a method, removing a field on a return model, changing a parameter type, broadening an exception's superclass, narrowing accepted vector dimensionality. |
| Non-breaking changes | Adding new optional keyword-only parameters, adding new methods, adding new fields to return models (Pydantic models are not `extra="forbid"` for consumers), adding new error subclasses under `RepositoryError`. |
| Static checking | mypy `--strict` on `app.repositories.pgvector`; CI fails on signature drift. |

### Version History

| Version | Changes | Backward Compatible |
|---------|---------|---------------------|
| 1.0 | Initial: `upsert_embeddings`, `search_similar`, `delete_by_document`, `count_by_tenant`. | N/A |

### Deprecation Flow

| Step | Action | Timeline |
|------|--------|----------|
| 1 | Mark method/field with `@deprecated` (PEP 702) and a docstring note. | Immediately |
| 2 | `warnings.warn(DeprecationWarning, stacklevel=2)` on first call per process. | Same release |
| 3 | Grep for callers in monorepo; open follow-up issues. | Same release |
| 4 | Remove after minimum 2 minor releases or 60 days, whichever is later. | |
