## 1. Motivation

Neo4j is the graph database backing the GraphRAG pipeline and contract relationship modeling (see `memory-bank/systemPatterns.md#ai-integration-patterns`). We are upgrading to the latest Neo4j 5.x patch release to:

- Pick up cumulative CVE fixes and bug fixes in the 5.x line.
- Align the `neo4j` Python driver with the server minor version (driver ↔ server compatibility matrix requires driver ≥ server minor).
- Refresh the APOC Core and Graph Data Science (GDS) plugin versions, which are tightly coupled to the server patch version.

**Triggering event**: Scheduled Phase 1 Week 1-2 infrastructure hardening; Neo4j image in `docker-compose.yml` was floating on an older 5.x tag without digest pinning.
**Tracking issue**: PR 1.5.1

## 5. Breaking Changes

Reviewed changelogs: Neo4j 5.15 → 5.26 server, `neo4j` Python driver 5.15 → 5.26, APOC 5.15 → 5.26, GDS 2.6 → 2.13.

| Package | Breaking Change | Impact on Our Code | Fix Required | Tracked In |
|---------|----------------|-------------------|-------------|------------|
| Neo4j server | Cypher planner deprecations: `CREATE` with pre-existing index behavior tightened; some `CALL { ... }` subquery implicit scoping rules stricter | Graph ingestion Cypher in `app/repositories/graph/` — 2 queries use implicit scoping | Add explicit `WITH` passthrough | PR 1.5.1 |
| Neo4j server | `dbms.security.procedures.unrestricted` now requires explicit allow-list for APOC procedures used by GDS | docker-compose env `NEO4J_dbms_security_procedures_unrestricted=apoc.*,gds.*` | Add allow-list entry | PR 1.5.1 |
| Python driver | `Result.single()` stricter: raises if >1 record instead of warning | ~4 call sites in `app/repositories/graph/` | Switch to `Result.single(strict=True)` explicitly or use `.fetch(1)` | PR 1.5.1 |
| Python driver | Deprecated `Neo4jDriver.close()` sync call in async context now raises | Verify async session lifecycle in `app/core/graph_db.py` | Already using `AsyncDriver.close()`; no change | — |
| APOC Core | Several `apoc.create.*` procs moved to APOC Extended (not installed) | Verified: we only use `apoc.load.json`, `apoc.periodic.iterate`, `apoc.merge.*` — all still in Core | None | — |
| GDS 2.13 | Graph projection native syntax removed (Cypher projection only) | Not yet used in Phase 1 | N/A | — |

**Impact summary**: ~6 files in `app/repositories/graph/` need minor updates; all covered by existing integration tests in `tests/repositories/test_graph_*.py`.

## 9. Rollback Plan

```bash
# Revert Python driver
pip install neo4j==5.15.0

# Revert Docker image (restore previous digest-pinned tag)
git revert <PR-1.5.1-commit>
docker-compose pull neo4j
docker-compose up -d neo4j

# Data compatibility: Neo4j 5.26 store format is forward-compatible within 5.x;
# downgrading from 5.26 → 5.15 requires restoring from pre-upgrade backup
# (taken automatically in the upgrade runbook before `docker-compose up`).
```

**Pre-upgrade backup**: `neo4j-admin database dump neo4j --to-path=/backups/pre-5.26-upgrade/` — verified in staging.
**Tracking issue for rollback verification**: PR 1.5.1 (rollback drill completed in staging)
